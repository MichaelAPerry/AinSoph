# ITEMS.md
## Ain Soph — Items

---

### WHAT AN ITEM IS

An item is anything physical in the world that is not a PC or NPC.

The ground is an item. The sky is an item. A tree is an item. An animal is an item. A plant is an item. A piece of manna is an item.

Items can be fixed in place or moveable. That distinction is handled by world physics, not the item schema.

---

### THE FIRST ITEM

Manna is the first and only food item at world start.

It is the one thing that sustains life before players and NPCs create anything else. Everything else is built.

Manna has a lifespan of 1 day. It is the model for all living item timers.

---

### ITEM TAXONOMY

| Type | Definition | Example |
|------|-----------|---------|
| **Primitive** | Exists at world start. Not created by anyone. | Ground, sky, manna |
| **Natural** | Generated by world rules — biomes, flora, fauna. | Tree, mushroom, animal |
| **Crafted** | Created by a PC or NPC. Passes through the Triune Council. | Chair, tool, weapon |

---

### NUTRITION

Every living character must eat once per day. This is binary — you ate or you didn't. There are no calories, no units beyond the daily requirement.

Any item with edible: true satisfies the eat survival requirement for one day when consumed.

---

### DEGRADATION

Living items have a lifespan. When the lifespan expires the item is gone.

Non-living items do not degrade.

The lifespan of a living item is defined in the item itself. It is set at creation for crafted items, and by world rules for natural items.

---

### ANIMALS

Animals are items that move and behave.

Animals are simpler than NPCs but share some of their structure. They have the four memory slots (will, thought, feeling, action) and the four primitives (move, see, hear, talk).

At world start, animals have the same four primitive actions as everything else. Further interactions — hunting, skinning, cooking, taming, riding — emerge when players or NPCs create those skills. The world does not anticipate them.

**Population:** Animals operate on a replacement rate. When one animal dies, two appear on the grid immediately adjacent to where it died. Animal populations self-replenish by design.

### THE ITEM SCHEMA

```json
{
  "id": "",
  "name": "",
  "type": "",
  "description": "",
  "created_by": "",
  "council_status": "",
  "properties": {
    "edible": false,
    "living": false,
    "lifespan_days": null,
    "animal": false,
    "notes": ""
  },
  "memory": {
    "will": "",
    "thought": "",
    "feeling": "",
    "action": ""
  },
  "primitives": {
    "move": true,
    "see": true,
    "hear": true,
    "talk": true
  },
  "lineage": ""
}
```

**Field definitions:**

| Field | Type | Notes |
|-------|------|-------|
| id | string | Unique identifier |
| name | string | What the item is called |
| type | string | primitive / natural / crafted |
| description | string | What the item is |
| created_by | string | Player ID or NPC ID. Empty for primitives and naturals. |
| council_status | string | pending / approved / rejected. Empty for primitives and naturals. |
| properties.edible | boolean | True if consuming this item satisfies the eat requirement for one day. |
| properties.living | boolean | True if this item has a lifespan timer. |
| properties.lifespan_days | number or null | How many real days before this item expires. Null for non-living items. |
| properties.animal | boolean | True if this item is an animal — it moves, behaves, and has memory and primitives. |
| properties.notes | string | Any additional properties defined by the creator. |
| memory | object | Only populated for animals. Same four slots as NPCs. Max 1024 chars each. |
| primitives | object | Only populated for animals. Each primitive can be true, broken, or lost. |
| lineage | string | Append-only record of who created and modified this item. |
